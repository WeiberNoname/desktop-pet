DESKTOP PET MASCOT V3 - SEQUENTIAL EXECUTION LOGIC FLOW

This document details the step-by-step sequential logic flow of the application
from launch to user interaction, multi-language rendering, and diagnostics.

================================================================================
STEP 1: INITIAL RUNTIME STARTUP (main.js)
================================================================================
1. The Windows OS launches `DesktopPet.exe` (or developer runs `npm start`).
2. Electron loads the main process script [main.js].
3. Hardware GPU switches run: `force-high-performance-gpu`, `ignore-gpu-blocklist`,
   and `disable-backgrounding-occluded-windows` are applied to prevent Chromium throttling.
4. Central diagnostics logger (`logDiagnostic`) initializes and opens `assets/diagnostics.log`.
5. Native Steamworks API is initialized via `@skyatnpm/steamworks-js` (App ID 480) with
   `MockSteamClient` fallback if Steam is offline.
6. Electron emits the `'ready'` event, triggering `createWindow()`:
   - Screen coordinates are queried via `screen.getPrimaryDisplay()`.
   - Dimensions default to 350x350, positioned at bottom-right corner shifted 50px from taskbar margins.
   - BrowserWindow is spawned with options: `transparent: true`, `frame: false`, `alwaysOnTop: true`,
     `resizable: false`, and `hasShadow: false`.
   - Background 20-minute uptime timer (`uptimeAchievementInterval`) starts monitoring for Steam achievement `ACH_TRAVEL_FAR`.
   - Window boundary edge check polling (`edgeCheckInterval`, 100ms) initializes to verify cursor bounds.
7. The window loads [index.html].
8. Initial window state: Click-through is enabled immediately by calling `mainWindow.setIgnoreMouseEvents(true, { forward: true })`.

================================================================================
STEP 2: WINDOW LOAD & MODULE RESOLUTION (index.html & i18nManager.js & renderer.js)
================================================================================
1. Chromium loads the HTML document structure of [index.html].
2. The HTML parser reads the `<script type="importmap">` tag.
   - Registers `"three"` to bare import Three.js ESM modules.
   - Registers `"i18next"` to expose `./node_modules/i18next/dist/esm/i18next.js`.
3. The browser imports `i18nManager.js` and initializes the i18next localization engine:
   - Evaluates system language via `detectSystemLanguage()` (`navigator.language`).
   - Loads translations asynchronously from `locales/<code >/translation.json` (31 supported locales).
   - Translates all DOM elements containing `data-i18n` and `data-i18n-title` attributes.
4. The parser loads `<script type="module" src="renderer.js"></script>` to execute [renderer.js].
5. Node integrations are imported into the renderer process (`ipcRenderer`, `fs`, `path`, `pathToFileURL`).
6. Once DOM ready, the `'DOMContentLoaded'` listener fires `init()`.

================================================================================
STEP 3: WEBGL ENGINE & SETTINGS SETUP (init() in renderer.js)
================================================================================
1. The DOM container `#container` is fetched.
2. The engine executes `readSettingsFile()`:
   - Scans `assets/settings` (or `settings.txt`).
   - Parses key-value pairs (`language`, `width`, `height`, `scale`, `fontSizeScale`, `bobbing`,
     `viewOnly`, `lockPosition`, `gpuOptimize`, `mouseOptimize`, `settingsLeft`, `spinX/Y/Z`, `speedX/Y/Z`).
   - If corrupted, heals settings automatically in memory and rewrites file cleanly.
3. If language is saved in settings, `changeLanguage(currentSettings.language)` updates UI locale.
4. Instantiates `THREE.Scene`, `PerspectiveCamera`, and transparent `THREE.WebGLRenderer({ alpha: true, antialias: true })`.
5. Appends WebGL canvas to DOM.
6. Sets up lighting (ambient, directional with shadows).
7. Triggers `detectAndLoadAsset()` to resolve active mascot character.
8. Runs `setupSettingsUI()` to bind gear button hover, dropdowns, sliders, view controls guide, and Diagnostics Console.

================================================================================
STEP 4: ZERO-CODE ASSET RESOLUTION & PREVIEW GENERATOR
================================================================================
1. `getAssetsPath()` locates active `assets/` directory.
2. `startBackgroundPreviewGenerator()` scans custom `.glb`/`.gltf` files in `assets/` and generates thumbnail images offscreen into `assets/.previews/`.
3. Populates mascot selection grid (`#model-select-grid`) with previews.
4. **CASE A: Default procedural mascot selected ('procedural')**
   - Calls `createMascot()` to build pink bunny (sphere body, ears, eyes, nose, cheeks).
5. **CASE B: Custom model asset selected**
   - Uses `GLTFLoader` to load model.
   - Calculates bounding boxes to center model and anchor base/feet flush with ground.
   - Parses embedded animations for idle ("idle", "stay", "breathe") vs interaction ("jump", "spin", "click", "react") clips.
   - Recalculates camera Z distance on save to auto-fit custom model inside viewport dimensions.

================================================================================
STEP 5: INTERACTION & BLENDER-STYLE VIEWPORT CONTROLS
================================================================================
1. Global mouse listeners (`mousemove`, `mousedown`, `mouseup`) handle raycasting hover hit-tests over mascot geometry.
2. Clicking mascot triggers interaction animation (360° spin & jump or react clip).
3. Dragging mascot or panel background moves Electron window across monitors (bypasses inputs/sliders).
4. Viewport 3D navigation controls:
   - Orbit (Rotate): `Alt + Left-Drag` or `MMB-Drag`.
   - Pan (Translate): `Shift + Left-Drag` or `Shift + MMB-Drag`.
   - Zoom (Depth): Scroll wheel or `Ctrl + Left-Drag`.
   - Orthographic views: Key `1` (Front), `3` (Right), `7` (Top).
   - View Reset: Key `9` (180° flip), Key `.` / `F` (Reset origin), `Alt + Double-Click` on mascot.
5. View-Only Mode (`viewOnly` checkbox or `Ctrl + V` shortcut) smoothly fades window opacity to `0.0` on hover, passing mouse input directly through to applications beneath.

================================================================================
STEP 6: DIAGNOSTICS & SYSTEM EVENTS
================================================================================
1. Speech bubble popups are purged; system alerts and config recovery messages output to Diagnostics Console (`logDiagnostic`).
2. Settings Panel collapsible "🔍 Diagnostics & Logs" console allows real-time viewing ("Refresh Logs") and purging ("Clear Logs") of system traces.
3. Clicking "Save Settings" saves parameters atomically to `assets/settings` and fires Steam achievement `ACH_HEAVY_RADAR`.
